
# Unified Public Safety Monitoring System (UPSMS)

UPSMS is an AI‑powered real‑time video analytics system that detects public safety threats – such as falls, fights, stalking, isolation, and gang formation – and automatically alerts authorities via email with video evidence.

## Features

- **Real‑time detection** from webcam or video files
- **Multi‑threat recognition**:
  - Fall detection (person horizontal for >5 seconds)
  - Harassment/conflict (two people close + erratic wrist movement)
  - Stalking (one person persistently behind another)
  - Isolation (person alone for a prolonged time)
  - Gang formation (multiple persons surrounding a single person)
- **Instant alerts**: saves incident clip, uploads to Supabase, and triggers n8n email with clip link
- **Configurable thresholds** via `config.py`
- **Robust error handling** – retries uploads and falls back to local storage on network failure
- **Optimized performance** – frame resizing for real‑time inference (~15‑20 FPS on laptop)

## Tech Stack

- **Python 3.10+**
- **YOLOv8** (Ultralytics) – object detection & pose estimation
- **OpenCV** – video capture & display
- **Supabase** – cloud storage & database
- **n8n** – workflow automation for email alerts
- **Gmail SMTP** – email delivery

## Prerequisites

- Python 3.10 or higher
- A Supabase account ([supabase.com](https://supabase.com))
- An n8n instance (cloud or self‑hosted) – you can use the provided cloud URL
- A Gmail account with 2‑factor authentication enabled (for app password)

Configuration
1. Supabase Setup
Create a Supabase project.

In SQL Editor, create the incidents table:

sql
CREATE TABLE incidents (
  id bigint generated by default as identity primary key,
  type text not null,
  severity text not null,
  location text not null,
  clip_url text not null,
  created_at timestamptz not null default now()
);
Create a storage bucket named incident-clips (make it public or configure permissions).

Get your project URL and anon key from Project Settings → API.

2. Configure Environment Variables
Copy .env.example to .env and fill in your credentials:

env
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
N8N_WEBHOOK_URL=https://your-n8n-instance.com/webhook/incident-webhook
3. n8n Workflow Setup
Import the provided workflow JSON (or create one as described in the project documentation).

Set the webhook URL (e.g., https://stalemartyr.app.n8n.cloud/webhook/incident-webhook).

Configure the Email node with your Gmail SMTP credentials (see below).

Activate the workflow.

4. Gmail App Password (for SMTP)
Since you have 2‑factor authentication enabled, generate an app password:

Go to Google App Passwords

Select Mail and Windows Computer, generate, and copy the 16‑character password.

In n8n's Email node, set:

Host: smtp.gmail.com

Port: 587

Secure: true

User: your full Gmail address

Password: the app password (with spaces)

5. Adjust Detection Thresholds (Optional)
All thresholds are in config.py. For quicker testing, you can lower the durations:

python
FALL_STATIC_DURATION_SEC = 2        # default 5
CONFLICT_PROXIMITY_DURATION_SEC = 3 # default 10
STALKING_DURATION_SEC = 3           # default 10
ISOLATION_DURATION_SEC = 3          # default 15
GANG_DURATION_SEC = 2               # default 5

Limitations & Future Work
Occlusion: In crowded scenes, pose estimation may fail. Tracking across frames helps but is not perfect.

Low light: YOLO performance degrades. Future: integrate thermal or IR cameras.

False positives: Can be reduced by further tuning thresholds or adding more heuristics (e.g., for stalking).

Multi‑camera support: Currently single stream; could be extended to multiple feeds.

Troubleshooting
Program crashes after incident: Ensure you have the latest detector.py with tuple‑subtraction fixes.

Upload fails: Check Supabase URL, anon key, and network connectivity. The system will fall back to incident_clips/ folder.

Email not sent: Verify n8n webhook URL and SMTP credentials (especially app password).
